SenchaTouch VSCode插件安装及使用

发表于 2019-09-25

介绍

SenchaTouch VSCode插件是在微软的Visual Studio Code编辑器中使用的扩展插件。该插件提供智能感知、代码检查、文档查找、以及与sencha cmd的集成。

扩展安装

离线安装

你需要去相关的门户网站下载Sencha VS Code plugin vscode-extjs-1.x.x.vsix。

选择VS Code编辑器中左侧的扩展标签。

点击...菜单按钮，选择从VSIX安装...,然后选择你下载的 vscode-extjs-1.x.x.vsix 文件进行安装。

Note:你也可以使用以下命令安装扩展:

在线安装

选择VS Code编辑器中左侧的扩展标签。

搜索”sencha”，选择Sencha Ext JS点击安装，安装完毕点击重新加载即可。

插件包安装(推荐)

找到VS Code的插件安装目录,Mac电脑为/Users/你的用户名/.vscode/extensions，windows电脑为C:\Users\ 你的用户名\.vscode\extensions，直接将插件包拖动到该目录下，重启VS Code编辑器，查看是否安装成功。

功能介绍

智能感知-代码提示

sencha VS Code plugin提供了强大的智能感知功能，它可以为所有的Ext JS和用户自定义类的配置项、方法和属性进行代码提示，当你在编辑Ext JS文件，只需要输入即可自动打开智能感知窗口进行代码提示。你也可以使用ctrl + space 的组合快捷键打开智能感知窗口。

配置智能感知功能

如果你的项目中包含.sencha目录，则默认情况下是启用智能感知的，或者当你使用Shencha Cmd创建的应用、工作空间或包时，Cmd会自动创建.sencha目录。

但是，如果你的项目不是使用Cmd创建的，你需要手动为你的项目启用智能感知功能。

  • 在你的VS Code 编辑器中按command + shift + p (ctrl + shift + p on windows)打开命令面板，输Enable Ext JS intelliSense for this Project，回车。

  • 这时可以看到项目中多了一个.sencha目录，打开.sencha/ide/config.json文件，配置项目的名称、项目根路径和SDK等。

Note: 注释仅做参考，不允许出现在代码中

代码导航

  • 将光标定位到您要导航的类名、属性、方法或事件上，您可以通过在按住command（ctrl on windows）键的同时单击它来导航到Ext JS框架或自定义类的源码中。
  • 您可以将光标移动到要打开定义的位置，然后按F12键，或右击-转到定义，在新的选项卡中打开源码。

  • 您还可以在当前的JavaScript选项卡中使用Option + F12（alt + F12 on windows）或右击-速览定义，在当前窗口中查看源码。

查找文档

Sencha Visual Studio Code插件还允许用户查看Sencha API文档。它支持Ext JS类名，xtypes和其他类型别名的文档查找。您还可以查找方法，属性，配置项，事件等。

将光标定位到相应的元素上，如类名、方法、属性等，然后按F1键。就可以在外部浏览器窗口中打开相应的Sencha API文档。

如果没有找到相关的API文档，默认会到当前应用所使用的SDK的根目录下查找API文档。

使用VS Code创建sencha 项目

  • 方式一：在VS Code资源管理其中右击 ，选择 “New Ext JS App”，然后完成向导中的步骤以创建Ext JS应用程序。
  • 方式二：在VS Code中使用Command + Shift + P组合键打开命令面板，键入“New”并选择“New Ext JS App”，然后完成向导中的步骤以创建Ext JS应用程序。

创建Ext JS应用程序向导。

  1. 指定应用的名称，回车。

  1. 指定本地存储路径，回车。

  1. 输入SDK路径，回车。

  1. 选择创建应用使用工具包，classic创建桌面应用项目，modern创建移动应用项目，universal创建 classic 和 modern 混合应用项目
